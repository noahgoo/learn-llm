Most LLM tokenizers descend from **Byte Pair Encoding (BPE)**: start from
an alphabet of single symbols, count all adjacent pairs in the training
corpus, merge the most frequent pair into a new vocabulary entry,
repeat to a target vocabulary size.<Cite id="sennrich2016" /> At
inference, text is segmented greedily by those same merge rules —
deterministic and reversible. That's why a leading space matters:
" the" and "the" are different vocabulary entries.

**Byte-level** BPE, introduced with GPT-2 and still standard, uses the
256 byte values as the base alphabet — any string in any language is
representable with zero out-of-vocabulary tokens.<Cite id="radford2019" />

The live model here is GPT-2 small: vocabulary **50,257** = 256 bytes
+ 50,000 learned merges + 1 end-of-text token, with merges forbidden
from crossing character categories.<Cite id="radford2019" /> Modern
frontier models keep the same algorithm with larger vocabularies
(~100k–250k entries) traded against sequence length. The output of this
stage — for any LLM — is just a sequence of integer IDs, shape `[seq]`.
